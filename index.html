Biografía

Taylor Alison Swift nació el 13 de diciembre de 1989 en Reading, Pensilvania, Estados Unidos. Desde pequeña mostró un gran interés por la música, especialmente por el country y el storytelling —la narración de historias a través de las canciones—, algo que más tarde se convertiría en su sello artístico.

Creció en una familia de clase media alta junto a sus padres, Andrea Gardner Swift y Scott Kingsley Swift, y su hermano menor Austin Swift. A los 11 años ya cantaba en concursos locales y eventos deportivos, e incluso interpretó el himno nacional de Estados Unidos en un partido de los Philadelphia 76ers.

Su pasión por la música country la llevó a aprender guitarra gracias a un técnico informático que le enseñó algunos acordes básicos. Con ese simple inicio, Taylor empezó a componer sus propias canciones, muchas inspiradas en sus experiencias personales, algo que más tarde se volvería una de sus características más admiradas.

Instalador de XAMMP

En este paso se instala y ejecuta el programa XAMPP, que permite crear un servidor local para el desarrollo de páginas web. Tras descargar e instalar el software desde la página oficial de Apache Friends, se abre el XAMPP Control Panel v3.3.0, donde se verifican los módulos principales (Apache y MySQL) y se confirma que el sistema está listo para el funcionamiento del sitio web.

Uso del Servidor Local

Para desarrollar la página web trabajé con XAMPP, un entorno local que me permitió ejecutar el sitio directamente desde mi computadora sin conexión a internet. Dentro de XAMPP activé el módulo Apache, que hizo posible visualizar los archivos HTML, CSS y PHP como si estuvieran alojados en un servidor real. Esto me permitió comprobar que las rutas, enlaces y formularios funcionaran correctamente antes de publicar la página. También pude probar los cambios en tiempo real sin afectar una versión en línea.

¿Qué es XAMPP?

XAMPP es un software que integra en un solo programa todo lo necesario para crear un entorno de servidor local. Su nombre se forma por:

  • X: Multiplataforma (funciona en Windows, Linux y macOS)
  • A: Apache (servidor web)
  • M: MySQL o MariaDB (gestor de bases de datos)
  • P: PHP (lenguaje de programación del lado del servidor)
  • P: Perl (lenguaje adicional)

Componentes Principales de XAMPP

  • Apache: Ejecuta los archivos web y los muestra en el navegador.
  • MySQL/MariaDB: Administra las bases de datos que usa el proyecto.
  • PHP: Interpreta el código de los archivos dinámicos.
  • phpMyAdmin: Permite gestionar bases de datos desde una interfaz visual.
  • FileZilla: Facilita la transferencia de archivos mediante FTP.
  • Mercury: Maneja servicios de correo electrónico.

Uso de Apache y MySQL

Dentro del panel de control de XAMPP activamos los servicios Apache y MySQL.

  • Al iniciar Apache, el servidor comenzó a interpretar los archivos web desde una carpeta, donde guardamos el proyecto.
  • Con MySQL, se pueden manejar bases de datos desde phpMyAdmin, aunque en este caso mi página no las requería.

Una vez ejecutado Apache, ingresé en el navegador la dirección: http://localhost/ seguida del nombre de mi carpeta del proyecto, y pude visualizar la página como si estuviera en línea. Este proceso me permitió asegurar que toda la estructura del sitio funcionara correctamente antes de subirla a una plataforma en la nube como Vercel.

Mainboard

La mainboard o también conocida como placa base, placa madre e o motherboard es la tarjeta del circuito que sirve de plataforma para la interconexión de un sistema operativo a la que se adaptan los demás componentes para que puedan funcionar una computadora de manera correcta. (Equipo editorial, Etecé, 2025b)

Se encuentran elementos del sistema, como el microprocesador, la RAM, las ranuras de expansión. En su interior, igualmente, se halla instalado el firmware del BIOS, software de sistema que permite regular y probar las funciones elementales del hardware y hace las veces de soporte para la carga del sistema operativo. (Equipo editorial, Etecé, 2025b)

Tamaño de la placa: La capacidad del hardware instalado está relacionada con el tamaño de la placa base y el espacio disponible. Es también identificado como factor de forma y de él depende la tipología de la placa que podamos instalar (Extended ATX, ATX, Micro ATX o Mini ITX). (Jvs-Informática & Aparicio, 2020) >
Características
  • Chipset: Es el centro de transferencia de información de todo el equipo, se divide generalmente en dos secciones diferentes:
  • Puente norte (northbridge): Interconecta la memoria RAM, el microprocesador y la unidad de procesamiento gráfico. (Equipo editorial, Etecé, 2025c)
  • Puente sur (southbridge): Interconecta los periféricos y los dispositivos de almacenamiento secundario, locales o externos. (Equipo editorial, Etecé, 2025c)

PARAMETROS:

SECURE BOOT

El Secure Boot o Arranque Seguro es una capa de seguridad instalada en el firmware UEFI de la placa base del ordenador, cuya función principal es impedir la ejecución de software de arranque no firmado o certificado antes de que el sistema operativo inicie. Esta característica es esencial, ya que permite bloquear ataques de malware, infecciones de virus y el uso de hardware no confiable que pueda comprometer el equipo, pero también afecta a sistemas operativos como las distribuciones GNU/Linux. (Fernández, 2019)

Históricamente, esta medida de seguridad ha generado polémica en la comunidad porque algunas distribuciones no están firmadas, lo que obligaba al usuario a deshabilitar la protección antes de instalar el sistema operativo. Sin embargo, no es un entorno cerrado: actúa como una autoridad de certificación (CA) que firma programas para organizaciones de confianza, y los usuarios tienen la opción de inscribir sus propias claves o eliminar las claves proporcionadas por la plataforma por completo. Actualmente, las distribuciones más importantes, como Ubuntu, Red Hat, Fedora, SUSE y Debian, han adaptado su firmware y dan soporte a Secure Boot (Otero, 2023).

Aunque el Secure Boot es un estándar UEFI, su soporte masivo comenzó con Windows 8, siendo esta la primera versión del sistema operativo en aprovechar la función. Actualmente, Windows 11 obliga a que esté activada. Sin embargo, muchos equipos compatibles tienen esta opción desactivada por defecto. En la práctica, esto requiere que el usuario acceda a la configuración de la UEFI (fuera de Windows) para habilitarla, un proceso que requiere seguir una serie de pasos concretos. Si estos pasos no se ejecutan correctamente, la secuencia de arranque puede interrumpirse, impidiendo el inicio normal del ordenador (López, 2025).

BOOT ORDER

El Orden de Arranque (Boot Order o Secuencia de Arranque) es la lista priorizada de dispositivos, establecida en el firmware del ordenador (la BIOS/UEFI), en los que la computadora buscará un sistema operativo.

Una vez completada la Autoprueba de Encendido (POST), el UEFI intenta arrancar desde el primer dispositivo de la lista. Si el dispositivo no es arrancable (o si es una unidad sin disco de instalación), lo omite y busca el sistema operativo en el siguiente elemento de la secuencia, y así sucesivamente. Aunque el disco duro es el dispositivo principal en circunstancias normales, la lista incluye otras opciones típicas como unidades ópticas (DVD), unidades USB (unidades flash) y recursos de red.

Por otro lado, no existe un orden único que se deba seguir siempre. El orden de los dispositivos de arranque debe modificarse cuando la situación lo requiere:

  1. Instalación o Mantenimiento: El orden debe cambiarse si se necesita arrancar desde un dispositivo distinto al disco duro. Por ejemplo, al instalar Windows o al usar un programa antivirus de arranque o de destrucción de datos, la unidad extraíble (DVD o USB) debe aparecer primero en la lista.
  2. Múltiples Discos: Si la computadora tiene varios discos duros instalados, es vital que el disco que contiene el sistema operativo esté primero en el Orden de Arranque para evitar que el UEFI se bloquee al intentar iniciar desde un disco que no es de arranque (Fisher, 2023).

Tecnologías de virtualización

La Tecnología de Virtualización es una capacidad del hardware que se activa desde el firmware (BIOS/UEFI). Permite a la computadora abstraer sus recursos físicos para que puedan ser compartidos y utilizados de forma nativa o directa por máquinas virtuales (VM), que son sistemas operativos virtuales que operan dentro del sistema operativo físico.

Los dos principales fabricantes de procesadores han desarrollado tecnologías específicas que se activan a través de la BIOS/UEFI:

  • Intel VT-x: Esta tecnología permite que el software de una máquina virtual se ejecute de forma nativa en una parte física dedicada del CPU y RAM. También puede abstraer recursos gráficos (GPU) y dispositivos de entrada/salida (E/S) mediante la tecnología TV-d.
  • AMD-V: Es la tecnología de AMD con características similares, diseñada para la optimización de recursos virtuales. Está diseñada para obtener la máxima compatibilidad con software de virtualización como Microsoft Hyper-V y cuenta con soporte optimizado para técnicas de virtualización de escritorios.

Software de virtualización compatibles con estas tecnologías

Para aprovechar esta tecnología, debe estar habilitada en la UEFI y el software de virtualización debe ser compatible. Los hypervisores (programas que crean y gestionan las VM) compatibles con VT-x o AMD-V incluyen:

  • VMware: Soporta VT-x, pero requiere activación manual.
  • VirtualBox: Es compatible con ambas tecnologías de Intel y AMD.
  • Microsoft Hyper-V: Soporta tanto AMD-V como VT-x para sus servicios de virtualización.
  • KVM y Xen: Son hypervisores de código abierto compatibles con ambas plataformas CPU.
  • Parallels: Es compatible con la tecnología VT-x de Intel.

Para verificar si un equipo es compatible con estas tecnologías, tanto Intel como AMD ofrecen software gratuito de diagnóstico que comprueba si nuestro equipo admite activar virtualización en BIOS o no (Castillo, 2018).
